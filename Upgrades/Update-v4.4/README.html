Guide upgrade to Laravel 8

Fundme Crowdfunding Platform

Step by Step

If you are downloading for the first time you should not perform these steps.
  • Requirements:
  • PHP >= 7.4.0
  • MySQL 5.6
  • BCMath PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Mbstring PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension
  • Ctype PHP Extension
  • JSON PHP Extension
  • cURL
  • Fileinfo PHP Extension
  • First steps
  • Create a folder on your server for example called _NEW
  • Upload the files found inside the Script folder to the folder you just created on your server _NEW
  • Do not delete the files that you will copy below because you will lose all the changes you have made.
  • You should only copy the files from your current site to the folders mentioned below.
IMPORTANT: If you have any questions to update or do not know how to do it, contact me before you start, once you have made the changes it can not be reversed. You must copy the files from your current site to the folder you created with the new version.
  • Root directory
    • You must copy the data of your App Name, App Url, database, email, captcha and social login in the .env file located in the _NEW folder
    IMPORTANT: If you have other variables in your .env file you must also copy them.
  • App Name
    • APP_NAME="Fundme"
  • App Url
    • APP_URL=http://localhost
    Don't put a slash at the end of the url
  • Database
    • DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=DB
DB_USERNAME=USER
DB_PASSWORD="PASS"
  • Mail Driver
    • MAIL_MAILER=sendmail
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS=no-reply@miguelvasquez.com
  • reCAPTCHA
    • INVISIBLE_RECAPTCHA_SITEKEY=YOU_KEY
INVISIBLE_RECAPTCHA_SECRETKEY=YOU_SCRETKEY
  • Set Timezone
    • TIMEZONE=UTC
  • Social Login
    • FACEBOOK_CLIENT_ID=
FACEBOOK_CLIENT_SECRET=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
    IMPORTANT: Place all data without spaces
  • app
    • Copy the Models folder of your current site
    • Copy the app/Http/Controllers folder of your current site
  • public
    • Copy all the folders from your current site
  • resources
    • Copy the folder resources/lang of your current site
    • Copy the folder resources/views of your current site
  • routes
    • Open the file routes/web.php of your current site, And put the following line use Illuminate\Support\Facades\Route; at the beginning of the file:
      • <?php

use Illuminate\Support\Facades\Route;
/*
|--------------------------------------------------------------------------
| Web Routes
|--------------------------------------------------------------------------
|
| This file is where you may define all of the routes that are handled
| by your application. Just tell Laravel the URIs it should respond
| to using a Closure or controller method. Build something great!
|
*/
    • After adding the line, save and copy to the _NEW/routes folder

Once all the folders and files have been copied, create a new folder called _backup there will move all the files of your current site as backup, after you move all the files, move all the files of the new folder where I copy the aforementioned files . There should not be any problem.

IMPORTANT: After copying all files, change the password of the database user.
IMPORTANT: If after updating you have a 500 error, make sure you have PHP 7.2 or higher, or if you have this error: The server requested authentication method unknown to the client... just change the password of the database user. My recommendation is to change the password of the database user to avoid problems.
